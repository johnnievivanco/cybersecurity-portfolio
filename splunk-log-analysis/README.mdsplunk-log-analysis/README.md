# Splunk Log Analysis

**Overview:**  
Simulated a SOC analyst investigation by analyzing system logs to identify suspicious activity and patterns.

**Tools Used:**  
- Splunk (SIEM tool)  
- Sample log files

**What I Did:**  
1. Uploaded and indexed sample log data in Splunk.  
2. Searched for failed login attempts and unusual activity.  
3. Filtered and analyzed abnormal IP addresses and login patterns.  
4. Documented findings and potential risks.

**Findings:**  
- Detected multiple failed login attempts from the same IP address.  
- Identified patterns consistent with a brute-force attack.

**Key Takeaways:**  
- Learned how to use a SIEM for real-time log monitoring.  
- Gained hands-on experience identifying security threats from logs.  
- Developed skills in documenting findings for both technical and executive audiences.
